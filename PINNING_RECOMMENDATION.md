# Pinning Recommendation

Recommended pinned repositories:

1. `digital-biosphere-architecture`
2. `persona-object-protocol`
3. `agent-intent-protocol`
4. `token-governor`
5. `fdo-kernel-mvk`
6. `aro-audit`

Rationale:

- These six repositories expose the theory hub plus the five canonical layers.
- This pinning restores visible continuity across persona, interaction, governance, execution integrity, and audit.
- It removes the need for the public front door to rely on a demo repository to explain the stack.

Not recommended for pinned slots:

- `verifiable-agent-demo`
- `agent-governance-benchmark`
- supporting annex repositories
- thin adapter repositories
- legacy lineage repositories

Explicit recommendation:

- `verifiable-agent-demo` should not continue to occupy a pinned slot.
- Supporting annexes and legacy repositories should not be pinned ahead of the five-layer core.
