# Three-Month Signaling Plan

## Objective

Focus public signaling on the repositories that best express the layered architecture and evidence-oriented research direction.

## Repositories to actively signal

- `digital-biosphere-architecture` — best repository for the full system explanation and canonical interpretation.
- `persona-object-protocol` — clearest conceptual entry into the stack through the persona layer.
- `verifiable-agent-demo` — smallest cross-layer walkthrough for people who want to see the architecture in practice.
- `aro-audit` — strongest evidence-oriented repository when rigor, reviewability, and conformance matter.

## Repositories to signal selectively

- `token-governor` — use selectively when the discussion is specifically about governance controls, budget policy, or runtime trade-offs.
- `agent-object-protocol` — use selectively when the discussion is specifically about structured object contracts or adjacent protocol work.

## Repositories not to foreground right now

Other repositories outside the current Digital Biosphere Architecture line should not be allowed to compete for front-page attention.

## Signal discipline

- Do not introduce a new top-level narrative casually.
- Point back to `digital-biosphere-architecture` when explaining the ecosystem.
- Use POP as the clearest concept entry.
- Use the demo when someone asks what this looks like in practice.
- Use ARO-Audit when rigor, evidence, or conformance comes up.
- Avoid letting side repositories dilute the main architecture story.
