- `redrock-opendpp-core`: predecessor from the EU DPP object and compliance branch; verifiable object concerns carry forward, but the repo is no longer part of the canonical core.

- `Kinetic-Robotics-FDO-Sovereignty`: adjacent historical branch for robotics sovereignty and kinetic digital objects; domain ideas remain, but the canonical core moved to a general five-layer architecture.
- `AASP-Core`: governance precursor from the adaptive agent sovereignty phase; governance intuitions survive, but the current line is layer-based rather than AASP-centered.
- `ISAS-Core`: predecessor from the industrial sovereignty implementation branch; lifecycle and anchoring ideas remain historically relevant, but the repo is not part of the current core.
